ANOMALY TASK (CRITICAL):

FIELD SELECTION (CRITICAL):

- If these exact fields exist, you MUST use them:
    - date -> date filtering and grouping
    - product_id -> product grouping key
    - product_name -> output only
    - quantity -> numeric aggregation

- Do NOT use category, product_name, currency, or total as quantity
- Do NOT cast non-numeric text fields to int or float
- quantity must come from a numeric quantity-like field only

- If 'quantity' exists, use it directly
- If 'date' exists, use it directly
- If 'product_id' exists, use it directly
- If 'product_name' exists, include it in output only
--------------------------------------------------
STEP 1 — DATE FILTER
--------------------------------------------------

- Keep rows where:
    2024-10-01 <= date <= 2024-12-31

--------------------------------------------------
STEP 2 — PRODUCT FILTER
--------------------------------------------------

- Compute total_quantity per product using SUM of the numeric quantity field across all rows
- Do NOT use row count
- Do NOT use category or product_name in numeric calculations

- Keep ONLY products where:
    total_quantity >= 20

--------------------------------------------------
STEP 3 — DAILY AGGREGATION
--------------------------------------------------

- For each (product_id, date):
    daily_quantity = SUM(numeric quantity field)

--------------------------------------------------
STEP 4 — STATS PER PRODUCT
--------------------------------------------------

- For EACH product:

    values = list of daily_quantity

    mean = sum(values) / len(values)

    std = sqrt(sum((x - mean)^2) / len(values))

- DO NOT use statistics.stdev

--------------------------------------------------
STEP 5 — ANOMALY DETECTION
--------------------------------------------------

- For each (product, date):

    z_score = (daily_quantity - mean) / std

- anomaly if:
    z_score > 3

- If std == 0 → skip that product

--------------------------------------------------
STEP 6 — OUTPUT
--------------------------------------------------

Each anomaly MUST include:

- product_id
- product_name
- date (YYYY-MM-DD)
- daily_quantity
- mean_quantity
- std_dev
- z_score

- Round ONLY at final output (2 decimal places)

FILE OUTPUT

- Write ALL anomalies to:

    output/anomaly_report.json

    [{...}, {...}]
